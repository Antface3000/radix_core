CHANGELOG
=========

All notable changes to Radix Core are documented here.

Format loosely based on Keep a Changelog. Versioning follows Semantic Versioning.


[UNRELEASED]
============


[0.2.1] - 2026-06-28
====================

Fixed
-----
- Lorebook / Story Bible UI in narrow dock: labels wrap, fields stack, list and
  form reflow instead of truncating.
- Generate and Save buttons use readable theme styles (no white-on-lime defaults).
- Active tab text stays dark on lime when switching Story Bible tabs.
- Embedded Lorebook and World State tabs drop duplicate headers inside Story Bible.
- Shorter lore field labels with tooltips for full descriptions.

[0.2.0] - 2026-06-28
====================

Added
-----
- VERSION file and version.json manifest; config reads VERSION at startup.
- src/updater.py - check for updates (git fetch or GitHub Releases API).
- update.bat and scripts/update.py - one-click update (git pull or release zip
  overlay; preserves data/, models/, .venv/, assets/piper/).
- scripts/release.py and RELEASE.txt - maintainer bump/tag/release workflow.
- Help panel About & Updates view with Check for updates and Update now.
- Startup update notification (once per day; dismiss or skip version).
- Story Bible auto-flush before agent runs (unsaved bible/world/lore saved).
- Agents panel Setting context preview (char count, pinned lore, Preview dialog).
- Shared bible field assembly in worldcontext (includes Synopsis).
- Story Bible resizable multiline fields (drag grip) with scroll when content
  overflows; runtime tab/segmented-button styling for readable selected tabs.

Changed
-------
- Documentation moved from .md to plain .txt (INSTALL.txt, USER_GUIDE.txt,
  README.txt, CHANGELOG.txt, and asset folder notes) - no markdown fences.
- run.py bootstraps its own virtual environment on first launch; no manual venv
  activation needed.
- Renamed run.bat -> Start Radix Core.bat; it runs scripts/start_services.py
  before opening the app.
- Default n_ctx raised to 8192 and max output tokens to 2400 (Settings -> Editor
  write_max_tokens / generation.max_tokens).
- Writing style preset labels: My Style, Alt Style, Neutral Style (Write bar).
- Marquee ... More overflow drawer on narrow windows; resizable side dock.
- Editor AI bar scrolls horizontally when the window is narrow.
- Projects launcher on startup replaces the old Welcome window.
- Setup, Settings, and Help removed from the left rail (marquee only).
- Orchestration planner, synthesis, and HITL liaison now receive the SETTING
  block when Inject setting is on.
- Story Bible Generate passes assembled setting context (not a placeholder label).
- Story Bible Save button shows * when there are unsaved changes.

Fixed
-----
- Startup crash: missing import config in editor_panel.py (style preset menu).
- Start Radix Core.bat now pauses on error so the console stays open if launch
  fails.
- Tab button contrast (inactive tabs readable) via CTkTabview / CTkSegmentedButton
  theme colors.
- Story Bible help text and scroll areas no longer clip in a narrow side dock.
- bind_scroll_width uses CTkScrollableFrame._fit_frame_dimensions_to_canvas (fixes
  startup AttributeError on CTk 5.x).
- Agents no longer miss Story Bible content when Orchestrate is used (planner
  and synthesis lacked setting injection).
- Synopsis field now included in agent/editor setting blocks.

[0.1.0] - 2026-06-27
====================

First public release. A local-only, editor-first creative writing suite that
drives local LLM agents (llama-cpp-python) plus ComfyUI image generation and
AllTalk/Piper voice - all on your own machine, no cloud APIs.

Added
-----
Editor and writing:
  - Editor-first workspace with rich-text tagging, word goal, Focus, Typewriter.
  - Write, Brainstorm, Chat, Visualize, Listen, Ask Agent on the selection.
  - Ghostwriter -> critics write pipeline with optional full-team pass.
  - Draft edits with Accept All / Reject All / Review.
  - Live lore auto-scan under the editor.

Agents and orchestration:
  - 14+ personas with global + per-project overrides.
  - Multi-agent orchestration with optional HITL.
  - Single-slot model loading for 8 GB GPUs; MOCK mode without models.

Story data:
  - Per-project workspaces; Story Bible, Lorebook, World State, Outline.
  - Lore capture markers ([[REMEMBER]], [[CHARACTER]], etc.).

Services hub:
  - Setup panel: ComfyUI, AllTalk, Piper connect/verify/sync.
  - Top-bar heartbeat dots; Sync Assets (copy-only).

Image and voice:
  - ComfyUI client; TTS via AllTalk or Piper.

UI:
  - Dark/lime theme; left rail + pop-out dock; Music panel (Deezer WIP).
  - Tooltips; in-app Help panel.

Docs and install:
  - install.bat, get_models.bat, run.py launcher.

Architecture:
  - WritingEngine over shared AgentEngine (single model slot).
